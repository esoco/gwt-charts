21  /**
22   * The ChartEditor class is used to open an in-page dialog box that enables a user to customize a visualization on the
23   * fly.<br>
24   * To use ChartEditor:
25   * <ul>
26   * <li>Load the {@link ChartPackage#CHARTEDITOR} package. You do not need to load the packages for the chart type that
27   * you render in the editor; the chart editor will load that package for you.</li>
28   * <li>Create a ChartWrapperObject object that defines the chart for the user to customize. This chart will be shown in the
29   * dialog, and the user uses the editor to redesign the chart, change chart types, or even change the source data.</li>
30   * <li>Create a new ChartEditor instance, and register to listen for the "ok" event. This event is thrown when the user
31   * clicks the "OK" button on the dialog. When received, you should call ChartEditor.getChartWrapper() to retrieve the
32   * user-modified chart.</li>
33   * <li>Call ChartEditor.openDialog(), passing in the ChartWrapperObject. This opens the dialog. The dialog buttons enable the
34   * user to close the editor. The ChartEditor instance is available as long as it is in scope; it is not automatically
35   * destroyed after the user closes the dialog.</li>
36   * <li>To update the chart in code, call setChartWrapper().</li>
37   * </ul>
38   */
