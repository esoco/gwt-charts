33  /**
34   * A ChartWrapper class is used to wrap your chart and handle all loading, drawing, and Datasource querying for your
35   * chart. The class exposes convenience methods for setting values on the chart and drawing it. This class simplifies
36   * reading from a data source, because you do not have to create a query callback handler. You can also use it to save a
37   * chart easily for reuse.
38   * 
39   * Another bonus of using ChartWrapper is that you can reduce the number of library loads by using dynamic loading.
40   * Additionally, you don't need to load the JSAPI library explicitly, and ChartWrapperObject will handle looking up the
41   * chart
42   * libraries for you, so you do not need to specify any chart libraries in your loading statement. See the examples
43   * below for details.
44   * 
45   * However, ChartWrapper currently only propagates a subset of events thrown by charts: select, ready, and error. Other
46   * events are not transmitted through the ChartWrapper instance; to get other events, you must call getChart() and
47   * subscribe to events directly on the chart handle.
48   * 
49   * Supports automatically resizing via {@link RequiresResize}, which means that all parents
50   * must implement {@link com.google.gwt.user.client.ui.ProvidesResize} for this to work.
51   * 
52   * @param <T> the options this chart implements
53   */
54  public class ChartWrapper<T extends Options> extends Widget implements RequiresResize {
